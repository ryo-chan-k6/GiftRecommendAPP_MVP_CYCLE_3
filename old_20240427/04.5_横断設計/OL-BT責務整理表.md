# 1. 前提整理

## 1-1. 事実

今回の区分ルールは、引継ぎメモのとおりです。

| 区分   | 定義                               |
| ------ | ---------------------------------- |
| OL     | ユーザー操作に同期して実行する処理 |
| BT     | 非同期・事前・定期で実行する処理   |
| Hybrid | OLで記録し、BTで集計・反映する処理 |

## 1-2. 推論

API仕様書 / バッチ仕様書に接続するには、**機能単位ではなく処理単位**で以下を明確化する必要があります。

- 誰が起点か（ユーザー / 定期実行 / 運用者）
- 応答待ちが必要か
- 事前計算できるか
- 再計算コストが高いか
- 即時反映が必要か

このため、今回は**「処理責務」単位**で整理します。
