## 1. 目的

本ドキュメントは、本サービスにおけるデータの保持期間および集約方針を定義し、

- ストレージ使用量の最適化
- レコメンド結果の再現性確保
- 品質監視・改善分析の継続性確保
- パフォーマンス維持

を両立することを目的とする。

---

## 2. 基本思想（最重要）

```
・全データを無制限に保持しない
・データ価値に応じて保持期間を分ける
・「生データ → 集約データ → 要約データ」の多層構造で管理する
・再現性が必要なデータは長期保持する
・再生成可能なデータは短期保持または削除する
・派生データでも「分析価値が高いもの」は中期保持する
```

---

## 3. データ分類（保持設計の前提）

| 区分               | 意味                 | 例                   |
| ------------------ | -------------------- | -------------------- |
| 業務正本           | 業務成立・再現に必須 | recommendation_run   |
| 管理正本           | 設定・version        | semantic_config      |
| 行動正本           | ユーザー行動の原本   | interaction_log      |
| ログ正本           | 観測ログ             | run_event_log        |
| 派生データ（分析） | 集約・評価           | offline_eval_result  |
| 派生データ（監視） | 分布・統計           | feature_distribution |
| キャッシュ         | 一時データ           | embedding            |

---

# 4. 保持期間ポリシー（全体）

```
・業務正本：無期限保持
・管理正本：無期限保持
・行動正本：中長期保持
・ログ正本：短期保持
・派生データ：用途別に中期〜長期保持
・キャッシュ：短期のみ
```

# 6. 集約方針

---

## 6.1 基本方針

```
・生ログは短期保持
・長期分析は集約データで行う
・粒度を落として保持期間を延ばす
```

---

## 6.2 集約構造

```
生データ（秒・イベント）
↓
時間単位（hourly）
↓
日次（daily）
↓
長期（weekly / monthly）
```

---

## 6.3 具体例

### interaction_log

```
interaction_log
↓
interaction_log_daily_summary
```

---

### metric_log

```
metric_log
↓
metric_log_hourly_summary
↓
metric_log_daily_summary
```

---

### 評価データ

```
offline_eval_result
↓
offline_eval_summary（将来）
```

# 7. 削除 / アーカイブ方針

---

## 7.1 削除対象

| 対象             | 条件     |
| ---------------- | -------- |
| run_event_log    | 30日超過 |
| metric_log（生） | 7日超過  |
| キャッシュ       | TTL超過  |

---

## 7.2 アーカイブ対象

| 対象                            | 方法                   |
| ------------------------------- | ---------------------- |
| interaction_log                 | 圧縮 or 外部ストレージ |
| error_log                       | 低頻度領域へ移動       |
| offline_eval_result（古いもの） | 要約化または外部保存   |

---

## 7.3 削除原則

```
・削除は必ず集約後に行う
・再現不能になる削除は禁止
```

---

# 8. パフォーマンス対策

```
・パーティション（日時単位）
・古いデータの削除
・集約テーブルの利用
・インデックスの最適化
```

# 9. トレードオフ

| 項目           | 長期保持 | 短期保持 |
| -------------- | -------- | -------- |
| 分析精度       | 高       | 低       |
| コスト         | 高       | 低       |
| パフォーマンス | 低下     | 良好     |

---

## 方針

```
・価値があるものだけ長期保持
・それ以外は集約で代替
```

```
・正本は長期保持
・ログは短期保持＋集約

・キャッシュは短期のみ

```
