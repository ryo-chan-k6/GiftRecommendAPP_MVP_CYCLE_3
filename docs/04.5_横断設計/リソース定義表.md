---
## 1. 目的

本ドキュメントは、本サービスにおける**リソース（API / DB / UI / ログ共通の業務単位）**を定義することを目的とする。

リソースは以下の設計の基準となる。

- API設計（URL / Path）
- データモデル（テーブル設計）
- 画面設計（UI単位）
- ログ設計（記録単位）
- バッチ設計（処理単位）
---

## 2. リソース定義

### 2-1. 定義

リソースとは、

```
API・DB・UI・処理において、識別・参照・生成・管理の対象となる業務上意味のある単位
```

とする。

---

### 2-2. 設計原則

| 原則           | 内容                                     |
| -------------- | ---------------------------------------- |
| 業務意味単位   | 技術単位ではなく業務単位で定義する       |
| 安定性         | 頻繁に意味が変わらない単位であること     |
| 責務分離       | 他リソースと責務が混在しないこと         |
| 層横断性       | API / DB / UI / ログで共通利用できること |
| 非アルゴリズム | 内部処理・計算はリソースにしない         |

---

## 3. リソース分類

| 分類   | 内容           |
| ------ | -------------- |
| 入力系 | ユーザー要求   |
| 実行系 | 処理単位       |
| 出力系 | 結果           |
| 中間系 | 候補・途中状態 |
| 管理系 | 設定・モデル   |
| 評価系 | human_eval     |
| 行動系 | interaction    |
| ログ系 | run_event 等   |

## 5. リソースに含めない対象

| 対象                | 理由       |
| ------------------- | ---------- |
| feature / embedding | 内部計算   |
| retrieval / ranking | 処理工程   |
| score分解           | 中間値     |
| lambda_ctx          | パラメータ |
