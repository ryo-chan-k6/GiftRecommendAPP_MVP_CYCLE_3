## 1. 目的

本ドキュメントは、本サービスにおける各機能・画面・API・バッチ処理が、どのデータ供給を前提として成立するかを整理することを目的とする。

本整理により、以下を明確化する。

- どのデータが正本か
- どのデータが派生か
- どのデータがキャッシュ供給前提か
- OLで参照するか、BTで供給するか
- 画面/API/BTがどのデータに依存するか
- 物理設計時に必要な正本テーブル・派生テーブル・キャッシュ対象

---

## 2. 基本方針

```
・正本データと派生データを分離する
・embedding はMVP時点では正本永続化しない
・embedding は再生成可能な派生データとして扱う
・user_embedding はOL生成 + 短期キャッシュを前提とする
・item_embedding はBT生成 + キャッシュ供給を前提とする
・重い集計・分布統計・監視用データはBT供給を前提とする
・画面/APIが必要とするデータは、参照元・供給元・更新タイミングまで明示する
```

## 4. 読み方の補足

### 4.1 データ区分

| 区分       | 意味                                       |
| ---------- | ------------------------------------------ |
| 正本       | 業務上の真の状態                           |
| 管理正本   | 設定・version等の真の状態                  |
| ログ正本   | append-only の記録データ                   |
| 派生       | 正本から生成される集約・統計・監視用データ |
| 基盤データ | SaaS/基盤側で自動収集されるメトリクス      |

---

### 4.2 供給方式

| 供給方式 | 意味                       |
| -------- | -------------------------- |
| OL生成   | API/Reco処理中に生成・保存 |
| OL参照   | API/Reco処理中に即時参照   |
| BT生成   | バッチで生成・更新         |
| 自動収集 | 基盤や監視SaaSが収集       |

---

### 4.3 保存方針

| 保存方針                        | 意味                            |
| ------------------------------- | ------------------------------- |
| 永続保存                        | DB等に正本/派生として保存       |
| 永続保存しない / 短期キャッシュ | Redis / メモリ等にTTL付きで保持 |
| 基盤側保持                      | 監視SaaSや実行基盤で保持        |
| 永続保存任意                    | MVPでは未保存も可、将来保存可能 |

---

## 5. embedding に関する重要前提

### 5.1 item_embedding

- **区分**: 派生データ
- **生成主体**: BT
- **保存方針**: テーブル化対象
- **利用方式**: テーブル（正本） ＋ キャッシュ（miss時のみDB）
- **理由**: embeddingは生成コストが高い。embeddingの入力（商品データ/モデル）の変更の度に再生成する必要があるが、入力の更新頻度は多くない想定のため、正本テーブルとして保持する
- キャッシュ無効化条件
  - item更新
  - semantic_config_version変更
  - embedding_model変更

### 5.2 user_embedding

- **区分**: 派生データ
- **生成主体**: OL
- **保存方針**: 正本永続化しない
- **利用方式**: 短期キャッシュ
- **理由**: リクエスト依存で毎回変わり、監査対象でもないため
- キャッシュ無効化条件
  - リクエスト条件変更（キー差分で対応）
  - semantic_config_version変更
  - embedding_model変更

### 5.3 使い分け

```
user_embedding = OL生成 + 短期キャッシュ
item_embedding = BT生成 + テーブル保存
```

## 6. OLで参照してよいデータ

以下は **リクエスト中に即時参照可能であることが前提**。

- semantic_config_version
- model_version
- item_feature
- normalization_stats
- item_embedding（キャッシュ）
- user_embedding（キャッシュまたは都度生成）
- recommendation_run/result（履歴参照時）

---

## 7. BT供給を前提とするデータ

以下は **OLで都度計算しない** 前提。

- item_feature
- item_embedding
- normalization_stats
- feature分布統計
- user feature分布統計
- 相互作用統計
- スコアバランス統計
- 監視集約
- 品質評価集約
- FB分類結果

---

## 8. 画面が直接参照しないデータ

以下は画面が直接参照するのではなく、管理API / 集約API経由で参照する。

- feature_distribution_summary
- user_feature_distribution_summary
- ranking_balance_summary
- recommendation_quality_summary
- metric_log_hourly_summary
- feedback_classification_result

---

## 10. 一言まとめ

```
本サービスでは、
「正本として永続管理すべきデータ」と
「派生としてBTまたはキャッシュで供給すべきデータ」を明確に分けることで、
低レイテンシ・再現性・品質監視・改善ループを両立する
```
