## 1. 目的

本資料は、Gift Recommendation Service における**主要サブシステム間のインターフェース境界**を整理することを目的とする。

本資料で扱うインターフェースは、以下を対象とする。

- Web ↔ Auth / API
- API ↔ Reco / Admin-Eval
- Batch ↔ 外部商品データ / DB
- 各アプリケーション ↔ 業務DB / ログ保存
- Manual運用 / 評価系IF

本資料は**概要設計レベルのIF一覧**であり、詳細なリクエスト項目、JSON Schema、SQL詳細は後続成果物で定義する。

---

## 2. IF区分

| IF区分     | 内容                                         |
| ---------- | -------------------------------------------- |
| 外部IF     | 利用者または外部システムが起点となるIF       |
| アプリ間IF | Web / API / Reco / Admin-Eval / Batch 間のIF |
| DB IF      | 業務DB、ログ保存先に対するRead / Write       |
| 手動IF     | 管理者・評価者など人手運用起点のIF           |

## 4. 再作成時の整理方針

### 4-1. 旧版から外したもの

旧版では、以下のような**Reco内部モジュール計算単位**までIFとして並んでいました。

- Relationship初期値取得IF
- Feature Distance計算IF
- Social/Symbolic集約IF
- Popularity算出IF
- Risk算出IF
- MMR多様化IF

これらは今後、**モジュール別入出力定義表** で持つのが適切です。

### 4-2. 今回の一覧に残したもの

今回残したのは、**サブシステム境界 / 永続化境界 / 手動運用境界** に立つIFです。

つまり、

- Web ↔ API
- API ↔ Reco
- Batch ↔ DB
- Admin/Eval ↔ DB
- 各処理 ↔ ログ保存

を中心に整理しています。
