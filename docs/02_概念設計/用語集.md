# 用語辞書 / 命名辞書

## Gift Recommendation Service

---

## 1. 基本方針

| 項目     | 内容                                                                                          |
| -------- | --------------------------------------------------------------------------------------------- |
| 目的     | 用語の意味・粒度・命名ルールを統一し、設計書・DB・API・実装での揺れを防ぐ                     |
| 対象     | 業務用語、論理名、物理名、スコア名、状態名、バージョン名                                      |
| 命名方針 | 論理名は英語スネークケース、APIは複数形リソース中心、DBテーブルは `apl.` 配下でスネークケース |
| 注意点   | signal / attribute / concept / feature / meaning を特に明確に区別する                         |

## 2.1 レコメンド全体概念

| 推薦結果商品 | recommendation_result_item | 最終的にユーザーに提示する個別商品 | Recommendation系         | candidate_item                   | resultの構成要素として明示追加              |

| 最終順位結果 | final_ranking_result       | 順位決定済み結果集合               | Ranking                  | recommendation_result            | ranking後の状態を明確化                     |

# 3. Gift Meaning モデル用語辞書

## 3.1 上位概念

| 日本語用語 | 英語論理名   | 定義                             | 備考              |
| ---------- | ------------ | -------------------------------- | ----------------- |
| 贈答意味   | gift_meaning | ギフトが持つ意味的価値の総称     | Social × Symbolic |
| 社会的意味 | social       | 礼儀性・安全性・無難さ寄りの意味 | 3次元で構成       |
| 象徴的意味 | symbolic     | 感情・特別感・関係性寄りの意味   | 5次元で構成       |

---

## 3.2 8次元 feature 定義

| 日本語名       | 英語論理名            | 区分     | 定義                                   | 補足                         |
| -------------- | --------------------- | -------- | -------------------------------------- | ---------------------------- |
| フォーマル度   | formality             | Social   | 礼儀性、形式性、きちんと感             | 高いほど改まった贈り物寄り   |
| 安全度         | safety                | Social   | 無難さ、失敗しにくさ                   | 高いほど万人向け             |
| ブランド適合性 | brand_appropriateness | Social   | ブランド面での妥当性、格との整合       | 高いほど対外的安心感が強い   |
| 感情性         | emotion               | Symbolic | 感情に訴える度合い                     | 高いほど気持ちが伝わりやすい |
| 新規性         | novelty               | Symbolic | 新しさ、意外性、珍しさ                 | 高いほど特別感が出やすい     |
| 親密性         | intimacy              | Symbolic | 相手との距離の近さを前提にできる度合い | 高いほど親しい関係向け       |
| 象徴性         | symbolic_identity     | Symbolic | 相手・関係・意味を象徴する強さ         | 高いほど意味づけが強い       |
| ストーリー性   | story_richness        | Symbolic | 背景や語れる文脈の豊かさ               | 高いほど物語性が高い         |

---

## 3.3 射影・集約関連

| 日本語用語   | 英語論理名    | 定義                            | 算出方針                                           |
| ------------ | ------------- | ------------------------------- | -------------------------------------------------- |
| social平均   | social_mean   | Social 3次元を集約した値        | `(formality + safety + brand_appropriateness) / 3` |
| symbolic平均 | symbolic_mean | Symbolic 5次元を集約した値      | 5次元平均                                          |
| 文脈重み     | lambda_ctx    | social寄り / symbolic寄りの重み | User Meaning から算出                              |

# 4. スコアリング用語辞書

| 日本語用語                      | 英語論理名       | 定義                                      | 主な利用箇所 | 注意点                                                                                              |
| ------------------------------- | ---------------- | ----------------------------------------- | ------------ | --------------------------------------------------------------------------------------------------- |
| 商品一致度内訳                  | item_match_score | 商品ごとのスコア構成全体                  |              | final_scoreとは別。Retrievalでの商品特性観点でのユーザー×商品コンテキストの一致度（コサイン類似度） |
| 特徴量一致度                    | feature_match    | feature距離から変換した一致度             |              | distanceとは別概念（原則の考え方は、1 - distance）                                                  |
| 文脈スコア                      | context_score    | user意図とitem意味の一致度（λ_ctx適用後） | Ranking      | 最終スコアの主成分                                                                                  |
| 社会的一致度                    | social_match     | userのsocial意図とitemのsocialの一致      | Matching     | social_mean系から導出                                                                               |
| 象徴的一致度                    | symbolic_match   | userのsymbolic意図とitemのsymbolicの一致  | Matching     | symbolic_mean系から導出                                                                             |
| 人気スコア                      | popularity_score | 市場人気やレビューに基づく補正値          | Ranking      | rank / review由来                                                                                   |
| リスクスコア                    | risk_score       | 贈答失敗リスクに関する補正値              | Ranking      | 高いほど減点側で使う設計が自然                                                                      |
| 最終スコア                      | final_score      | 表示順位決定に使う総合スコア              | Ranking      | `context + popularity - risk`                                                                       |
| ranking専用のスコアとして定義。 |
| 特徴量距離                      | feature_distance | user/item feature 間の距離                | Matching     | 途中計算用、最終表示スコアではない                                                                  |

# 9. 命名禁止・非推奨ルール

| 非推奨          | 理由                  | 推奨                                                 |
| --------------- | --------------------- | ---------------------------------------------------- |
| `data`          | 意味が広すぎる        | `signal`, `summary`, `result`, `stat`                |
| `info`          | 何の情報か曖昧        | `metadata`, `detail`, `summary`                      |
| `value`         | 文脈がない            | `final_score`, `mean_value`, `delta_value`           |
| `result` の乱用 | 何の結果か不明瞭      | `concept_result`, `offline_eval_result` のように限定 |
| `feature` 単独  | rawかnormalizedか不明 | `feature_raw`, `feature_normalized`                  |
| `score` 単独    | 何スコアか不明        | `context_score`, `risk_score`, `final_score`         |

# 11. 用語辞書の運用ルール

| 項目                             | ルール                                        |
| -------------------------------- | --------------------------------------------- |
| 新用語追加                       | 先に本辞書へ追加してから設計書へ反映する      |
| 命名変更                         | DB/API/資料への影響範囲を確認してから変更する |
| 日本語名と英語名                 | 1対1対応を原則にする                          |
| 類義語の扱い                     | 代表語を1つ決め、他は別名として備考に寄せる   |
| feature名                        | 8次元は固定し、別名を増やさない               |
| signal/attribute/concept/feature | この階層区分は崩さない                        |
