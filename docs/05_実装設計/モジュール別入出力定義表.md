## 1. 目的

本資料は、各モジュールについて以下を明確化することを目的とする。

- モジュールの主責務
- 実行契機
- 主な入力
- 主な出力
- 主な参照データ
- 主な更新データ
- OL/BT区分

本資料は、後続の

- API仕様書
- バッチ仕様書
- 画面設計書
- 実装

の基礎資料とする。

## 3. 使い方

この表は、次の成果物に分解して使います。

| 後続成果物                    | 何に使うか                                            |
| ----------------------------- | ----------------------------------------------------- |
| API仕様書                     | Auth / Web / API / Reco 入口系の詳細化                |
| バッチ仕様書                  | 商品データ収集、特徴量、embedding、評価バッチの詳細化 |
| 画面設計書 / ワイヤーフレーム | Web/UI制御モジュールと表示/操作の接続整理             |
| テーブル一覧 / DDL            | 主Read / 主Write の永続化対象整理                     |
| CI/CDパイプライン設計書       | DevOps系モジュールの具体化                            |

---

## 4. 補足

### 事実

- この表は「モジュール内部処理のアルゴリズム詳細」までは持っていません。
- ここではあくまで、**モジュール責務単位の入出力境界** を定義しています。

### 推論

- そのため、実装コードに近いレベルの詳細仕様は、
  - API仕様書
  - バッチ仕様書
  - 画面設計書
  - DDL
    に委ねるのが自然です。
