# プロジェクト工程定義

| Phase | 工程名               | 概要                                                             |
| ----- | -------------------- | ---------------------------------------------------------------- |
| P1    | 事業構想             | 事業目的・提供価値・顧客課題を定義する                           |
| P2    | ドメイン探索         | 業務領域、主要概念、業務ルール、関係者の言葉を整理する           |
| P3    | ドメイン要件定義     | ユースケース、機能要件、非機能要件、制約を定義する               |
| P4    | ドメインモデル設計   | ユビキタス言語、概念モデル、境界づけられたコンテキストを整理する |
| P5    | アプリケーション設計 | ユースケース、モジュール、API、データ、バッチ、外部IFを設計する  |
| P6    | 実装設計             | クラス、関数、DB物理設計、処理詳細、エラーハンドリングを設計する |
| P7    | 開発                 | 実装・単体テストを行う                                           |
| P8    | 結合・総合テスト     | API連携、画面連携、E2E、バッチ連携を検証する                     |
| P9    | リリース             | 本番反映、初期運用確認を行う                                     |
| P10   | 運用・改善           | フィードバックをもとにモデル・機能・運用を改善する               |
