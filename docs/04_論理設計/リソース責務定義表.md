# 1. リソース責務定義表の見方

| 項目           | 意味                           |
| -------------- | ------------------------------ |
| 主責務         | この概念が担う中心責任         |
| 持つべき情報   | この概念に所属させるべきデータ |
| 持たない情報   | 他概念に任せるべきデータ       |
| 境界のポイント | 混同しやすい概念との切り分け   |

---

# 2. 入力・実行・出力系

## 2.1 user_request（推薦要求）

| 項目           | 内容                                                                         |
| -------------- | ---------------------------------------------------------------------------- |
| 主責務         | ユーザーの1回の推薦依頼を表す                                                |
| 持つべき情報   | 入力全文、依頼時刻、依頼単位識別子                                           |
| 持たない情報   | 分解済み条件、実行状態、結果                                                 |
| 境界のポイント | `user_request_condition` は依頼の中身の明細、`recommendation_run` は実行単位 |

---

## 2.2 user_request_condition（推薦条件）

| 項目           | 内容                                                       |
| -------------- | ---------------------------------------------------------- |
| 主責務         | 推薦要求を条件単位に分解して保持する                       |
| 持つべき情報   | relationship, occasion, budget, desired, ng などの条件明細 |
| 持たない情報   | 実行状態、スコア、結果                                     |
| 境界のポイント | `user_request` が親、こちらはその構造化明細                |

---

## 2.3 recommendation_run（推薦実行）

| 項目           | 内容                                                                                  |
| -------------- | ------------------------------------------------------------------------------------- |
| 主責務         | 1回の推薦処理単位の管理                                                               |
| 持つべき情報   | request参照、run状態、開始/終了時刻、使用した semantic_config_version / model_version |
| 持たない情報   | 商品別スコア、分布統計、最終結果明細                                                  |
| 境界のポイント | `recommendation_result` は出力、`item_match_score` は計算詳細                         |

---

## 2.4 candidate_item（候補商品）

| 項目           | 内容                                                                 |
| -------------- | -------------------------------------------------------------------- |
| 主責務         | retrieval段階で残った候補商品を表す                                  |
| 持つべき情報   | run参照、item参照、候補入り理由や取得順序など候補集合情報            |
| 持たない情報   | 最終順位、最終表示理由                                               |
| 境界のポイント | `recommendation_result_item` は最終提示、`candidate_item` は途中候補 |

---

## 2.5 recommendation_result（推薦結果）

| 項目           | 内容                                  |
| -------------- | ------------------------------------- |
| 主責務         | runが返した結果集合を保持する         |
| 持つべき情報   | run参照、結果件数、生成時刻           |
| 持たない情報   | 個別商品の詳細スコア内訳              |
| 境界のポイント | `recommendation_result_item` の親集合 |

---

## 2.6 recommendation_result_item（推薦結果商品）

| 項目           | 内容                                                 |
| -------------- | ---------------------------------------------------- |
| 主責務         | ユーザーに提示する最終商品明細を保持する             |
| 持つべき情報   | result参照、item参照、表示順位、最終スコア、表示理由 |
| 持たない情報   | 全候補比較情報、featureごとの距離                    |
| 境界のポイント | `item_match_score` は内部計算、こちらは最終出力      |

---

# 3. User Meaning 系

## 3.1 external_feature（外部条件特徴量）

| 項目           | 内容                                              |
| -------------- | ------------------------------------------------- |
| 主責務         | relationship / occasion / pair 由来の特徴量を表す |
| 持つべき情報   | 外部条件起点の feature 値                         |
| 持たない情報   | 嗜好/NG由来特徴量、最終統合値                     |
| 境界のポイント | `internal_feature` と分ける                       |

---

## 3.2 internal_feature（内部条件特徴量）

| 項目           | 内容                               |
| -------------- | ---------------------------------- |
| 主責務         | 嗜好・NG由来の特徴量を表す         |
| 持つべき情報   | hint や NG 条件から得た feature 値 |
| 持たない情報   | 外部条件 feature、統合後 feature   |
| 境界のポイント | `external_feature` と責務分離      |

---

## 3.3 user_feature_raw（ユーザー生特徴量）

| 項目           | 内容                                                        |
| -------------- | ----------------------------------------------------------- |
| 主責務         | 外部＋内部統合後、正規化前のユーザー特徴量を保持する        |
| 持つべき情報   | 8次元 feature の raw 値                                     |
| 持たない情報   | 正規化後値、meaning射影                                     |
| 境界のポイント | 比較可能ではない。比較には `user_feature_normalized` を使う |

---

## 3.4 user_feature_normalized（ユーザー正規化特徴量）

| 項目           | 内容                                       |
| -------------- | ------------------------------------------ |
| 主責務         | 比較用に正規化したユーザー特徴量を保持する |
| 持つべき情報   | 正規化後 8次元 feature                     |
| 持たない情報   | raw値、最終スコア                          |
| 境界のポイント | `user_feature_raw` とは変換前後の違い      |

---

## 3.5 user_meaning_projection（ユーザー意味射影）

| 項目           | 内容                                           |
| -------------- | ---------------------------------------------- |
| 主責務         | ユーザーfeatureを social / symbolic に射影する |
| 持つべき情報   | user_social, user_symbolic                     |
| 持たない情報   | feature raw/normalized 全体                    |
| 境界のポイント | 射影結果であり、特徴量そのものではない         |

---

## 3.6 lambda_ctx（文脈重み）

| 項目           | 内容                                         |
| -------------- | -------------------------------------------- |
| 主責務         | user meaning から文脈重みを表す              |
| 持つべき情報   | λ_ctx 値                                     |
| 持たない情報   | social/symbolic 自体                         |
| 境界のポイント | `user_meaning_projection` から派生した計算値 |

---

## 3.7 preferred_context / non_preferred_context

| 項目           | 内容                                                  |
| -------------- | ----------------------------------------------------- |
| 主責務         | embedding生成用の検索文脈を表す                       |
| 持つべき情報   | preferred / non-preferred のコンテキスト文            |
| 持たない情報   | embeddingベクトル                                     |
| 境界のポイント | context はテキスト的中間表現、embeddingはベクトル表現 |

---

## 3.8 preferred_embedding / non_preferred_embedding

| 項目           | 内容                                                     |
| -------------- | -------------------------------------------------------- |
| 主責務         | retrievalに使うユーザー側ベクトルを保持する              |
| 持つべき情報   | embeddingベクトル                                        |
| 持たない情報   | rawテキストコンテキスト                                  |
| 境界のポイント | `preferred_context` / `non_preferred_context` の変換結果 |

---

# 4. Item Meaning / Retrieval 系

## 4.1 item_signal（商品シグナル）

| 項目           | 内容                                          |
| -------------- | --------------------------------------------- |
| 主責務         | 商品から抽出した推定材料を保持する            |
| 持つべき情報   | text / metadata / image / review 由来のsignal |
| 持たない情報   | 統合後属性サマリ                              |
| 境界のポイント | 原材料であり、統合済み表現ではない            |

---

## 4.2 item_attribute_summary（商品属性サマリ）

| 項目           | 内容                                              |
| -------------- | ------------------------------------------------- |
| 主責務         | 複数signalを統合した中間表現を保持する            |
| 持つべき情報   | 商品特性の統合サマリ                              |
| 持たない情報   | embeddingベクトル、最終feature                    |
| 境界のポイント | concept/feature推定とitem_context生成の入力になる |

---

## 4.3 item_context（商品コンテキスト）

| 項目           | 内容                                              |
| -------------- | ------------------------------------------------- |
| 主責務         | retrieval / embedding用の商品表現を保持する       |
| 持つべき情報   | embedding入力となる商品コンテキスト               |
| 持たない情報   | embeddingベクトル、feature値                      |
| 境界のポイント | `item_attribute_summary` から生成される検索用表現 |

---

## 4.4 item_embedding（商品Embedding）

| 項目           | 内容                                                             |
| -------------- | ---------------------------------------------------------------- |
| 主責務         | candidate retrieval に使う商品側ベクトルを保持する               |
| 持つべき情報   | item embedding ベクトル                                          |
| 持たない情報   | item contextテキスト、feature値                                  |
| 境界のポイント | retrievalで preferred / non_preferred embedding と比較される対象 |

---

## 4.5 item_concept（商品コンセプト）

| 項目           | 内容                                 |
| -------------- | ------------------------------------ |
| 主責務         | 商品に付与された意味ラベルを保持する |
| 持つべき情報   | conceptラベル、抽出根拠              |
| 持たない情報   | feature数値                          |
| 境界のポイント | conceptはラベル、featureは数値       |

---

## 4.6 item_feature_raw / item_feature_normalized / item_meaning_projection

| 概念                    | 主責務              | 持つべき情報               | 持たない情報          | 境界のポイント |
| ----------------------- | ------------------- | -------------------------- | --------------------- | -------------- |
| item_feature_raw        | 正規化前feature保持 | raw feature                | normalized値、meaning | raw段階        |
| item_feature_normalized | 比較用feature保持   | normalized feature         | raw値、meaning        | 比較用         |
| item_meaning_projection | social/symbolic射影 | item_social, item_symbolic | raw/normalized全体    | 射影結果       |

---

# 5. Matching / Ranking 系

## 5.1 feature_distance / feature_match

| 概念             | 主責務              | 持つべき情報        | 持たない情報          | 境界のポイント    |
| ---------------- | ------------------- | ------------------- | --------------------- | ----------------- |
| feature_distance | feature距離の保持   | featureごとの距離   | social/symbolic集約値 | 中間計算          |
| feature_match    | feature一致度の保持 | featureごとの一致度 | 最終スコア            | 距離→一致度変換後 |

---

## 5.2 social_match / symbolic_match

| 項目           | 内容                                   |
| -------------- | -------------------------------------- |
| 主責務         | 軸ごとの一致度集約を保持する           |
| 持つべき情報   | social_match / symbolic_match          |
| 持たない情報   | 最終スコア                             |
| 境界のポイント | `context_score` の入力になる中間集約値 |

---

## 5.3 context_score / popularity_score / risk_score / item_match_score / final_score

| 概念             | 主責務                   | 持つべき情報      | 持たない情報       | 境界のポイント |
| ---------------- | ------------------------ | ----------------- | ------------------ | -------------- |
| context_score    | 文脈一致の主成分         | λ適用後スコア     | popularity/risk    | 主スコア       |
| popularity_score | 人気補正                 | popularity値      | context/risk       | 補助           |
| risk_score       | リスク補正               | risk値            | context/popularity | 減点系         |
| item_match_score | 商品ごとのスコア内訳全体 | 各scoreのまとまり | 最終表示順位       | 計算結果全体   |
| final_score      | 順位決定用総合スコア     | final_score       | 表示理由           | ranking入力    |

---

# 6. 記録 / 計測 / 統計 系

## 6.1 phase_log / error_log / view_log / click_log

---

# 8. ルール / 定義 / バージョン管理 系

## 8.1 relationship_rule / occasion_rule / pair_rule / hint_dictionary / concept_definition / feature_definition / concept_feature_rule

| 概念                 | 主責務                     |
| -------------------- | -------------------------- |
| relationship_rule    | relationship→feature初期値 |
| occasion_rule        | occasion→feature初期値     |
| pair_rule            | relationship×occasion補正  |
| hint_dictionary      | 入力語→concept             |
| concept_definition   | conceptの意味定義          |
| feature_definition   | 8次元feature定義           |
| concept_feature_rule | concept→feature変換        |

---

## 8.2 semantic_config_version

| 項目           | 内容                                                                        |
| -------------- | --------------------------------------------------------------------------- |
| 主責務         | 意味推定ロジックの構成バージョンを管理する                                  |
| 持つべき情報   | concept / dictionary / rule / normalization / projection の採用バージョン群 |
| 持たない情報   | final_score式、ranking重み                                                  |
| 境界のポイント | 「意味の作り方」を固定する。`model_version` とは分離                        |

---

## 8.3 model_version

| 項目           | 内容                                                                       |
| -------------- | -------------------------------------------------------------------------- |
| 主責務         | スコアリング・ランキングの構成バージョンを管理する                         |
| 持つべき情報   | feature比較式、match集約式、context/popularity/risk/final score の式や係数 |
| 持たない情報   | concept辞書、feature定義そのもの                                           |
| 境界のポイント | 「順位の決め方」を固定する。`semantic_config_version` とは分離             |

---

# 9. 一言でまとめると

- **semantic_config_version**
  = 入力や商品情報を「意味」に変える責任
- **model_version**
  = 作られた意味を使って「順位」に変える責任
