# 機能概要

## F01. User Input Parse

### 役割

ユーザーが入力した贈答条件を受け取り、後続処理で使いやすい構造に整形する。

### 主な入力

- relationship
- occasion
- 好みの特徴
- 避けたい特徴
- 予算
- NG条件

### 主な出力

- 正規化済み入力データ
- 欠損補完済みの推薦リクエスト

### 処理例

- 入力値のバリデーション
- relationship / occasion のコード化
- 予算範囲の正規化
- NG条件の構造化（形態素解析などで実施を想定）

### 主な入力

- 正規化済みレコメンド条件
- 推定用変換テーブル
  - relationship rule
  - occasion rule
  - pair rule
  - hint dictionary
  - semantic concept

### 主な出力

- context_feature
- context_social
- context_symbolic
- hint_feature
- hint_social
- hint_symbolic
- user_social
- user_symbolic
- λ_ctx

### 内部ロジック

- relationship / occasion による基本feature付与
- pair rule による補正
- hint / semantic concept による補正
- social / symbolic 集約
- λ_ctx 算出
  - `λ_ctx = user_symbolic / (user_social + user_symbolic + ε)`

## F05. Gift Meaning Matching

### 役割

候補商品について、ユーザーと商品のGift Meaning Featureの一致度を詳細計算する。

### 主な入力

- user feature（social / symbolic）
- candidate items
- item feature（social / symbolic）
- λ_ctx

### 主な出力

- featureごとの match
- social_match
- symbolic_match
- context_score

### 計算例

- `distance_f = |user_f - item_f|`
- `match_f = 1 - distance_f`
- `social_match = mean(match_f)`
- `symbolic_match = mean(match_f)`
- `context_score = ((1 - λ_ctx) * social_match) + (λ_ctx) * symbolic_match`

## F06. Final Ranking

### 役割

Matching結果に popularity / risk を加味し、最終順位を決定する。

pupularityは加点要素、riskは減点要素の最終スコア計算の補正値として利用する。

- popularity：人気であること（レビュー件数が多い、評価が高い）
- risk：人気でないこと（レビュー件数が少ない、評価が低い）

risk補正値は、閾値を設定し下記の制御を検討する。

- 閾値を超える場合：その他のスコア要素に関わらず、レコメンド候補から除外する
- 閾値を超えない場合：最終スコア計算時の減点補正値として利用する。

### 主な入力

- context_score
- 商品レビュー評価情報
  - レビュー件数
  - レビュー評価

### 主な出力

- 補正値
  - popularity_adjust
  - risk_penalty_adjust
- `final_score`
- 商品順位

### 主な要素

- review_rating
- review_count
- ranking
- 一般評価性

## F11. Item Meaning Extraction

### 役割

商品テキストやメタ情報から、item meaning を生成する。

夜間のバッチ処理。

### 主な入力

- テキスト情報
  - 商品名
  - 商品説明
  - キャッチコピー
- 商品メタデータ
  - ジャンル
  - タグ
- 画像データ
  - CLIPなどによる画像解析にてテキスト情報化を検討したい。
- レビュー評価情報
  - レビュー件数
  - レビュー評価
  - ※レビューコメントは利用規約上利用不可のため対象外とする。
- semantic concept 定義
  - キーワード ⇔ Gift Meaning Featureを紐づける中間層
  - キーワード ⇔ Semantic Concept ⇔ Gift Meaning Feature
- hint dictionary
  - キーワード ⇔ semantic conceptの対応表
- ルール
  - semantic concept ⇔ Gift Meaning Featureの対応表
- LLM / embedding
- meaning_version

### 主な出力

- 商品特性情報（item_attribute）
- item_feature
- item_social
- item_symbolic

### 処理内容

- 商品特性情報解析
  - テキストデータ解析
  - メタデータ解析
  - 画像データ解析
  - レビュー評価解析
- semantic conceptスコア推定
- feature算出
- social / symbolic 集約
